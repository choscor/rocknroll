# Database Design

## Workflow

1. Identify entities, relationships, and access patterns.
2. Define schema fields, types, and constraints.
3. Plan indexes for common queries.
4. Consider migrations and backward compatibility.
5. Document any trade-offs.

## Output Expectations

Provide:
- Schema or model changes
- Migration considerations
- Query/index notes
