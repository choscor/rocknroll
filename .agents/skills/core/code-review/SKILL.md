# Code Review

## Workflow

1. Confirm the review scope (files, diff, or feature area).
2. Read the changes and key related files.
3. Check for correctness, security, and performance risks.
4. Verify API/contract consistency and naming.
5. Assess test coverage and suggest additions.
6. Summarize findings by severity.

## Output Expectations

Provide:
- Scope reviewed
- Findings grouped by severity
- Testing recommendations
- Overall risk assessment
