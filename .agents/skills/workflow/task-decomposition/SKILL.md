# Task Decomposition

## Workflow

1. Restate the goal in one sentence.
2. Identify constraints (time, scope, files, dependencies).
3. List assumptions that must hold.
4. Decompose into 3-10 ordered steps.
5. Call out risks or unknowns per step.
6. Define validation: tests or manual checks.

## Output Expectations

Provide:
- Short goal summary
- Numbered steps
- Risks/unknowns
- Validation plan
