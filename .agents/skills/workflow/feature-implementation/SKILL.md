# Feature Implementation

## Workflow

1. Clarify requirements and acceptance criteria.
2. Identify impacted modules and existing patterns.
3. Propose a minimal design and file changes.
4. Implement feature incrementally.
5. Add or update tests for new logic.
6. Validate with project scripts or manual checks.

## Notes

- Prefer extending existing modules over new ones.
- Keep changes scoped to requested behavior.

## Output Expectations

Report:
- Feature summary
- Files changed
- Tests run (or why skipped)
