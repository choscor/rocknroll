# Commit Message Writing

## Workflow

1. Identify the primary change and scope.
2. Use the format: `type(scope): short description`.
3. Keep it under 72 characters when possible.
4. Avoid unrelated details or multiple scopes.

## Output Expectations

Provide:
- Proposed commit message
- Optional alternatives (1-2)
