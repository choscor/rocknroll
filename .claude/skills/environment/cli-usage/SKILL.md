# CLI Usage

## Workflow

1. Prefer existing project scripts over ad-hoc commands.
2. Avoid destructive commands unless approved.
3. Capture key outputs and errors.
4. Re-run with elevated permissions only if required.
5. Summarize command results in plain language.

## Notes

- Keep command output concise in summaries.
- If a command cannot run, state why and suggest alternatives.

## Output Expectations

Report:
- Commands run
- Result summaries
- Any follow-up steps
