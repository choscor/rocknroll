# Dependency Management

## Workflow

1. Check if existing utilities cover the need.
2. Evaluate stability, license, and maintenance.
3. Prefer minimal, well-supported dependencies.
4. Document why the dependency is required.
5. Update lockfiles and run relevant checks.

## Output Expectations

Provide:
- Dependency decision and rationale
- Files updated
- Tests/commands run
