# System Design

## Workflow

1. Clarify goals, scale, and constraints.
2. Identify components and responsibilities.
3. Define interfaces and data flow.
4. Call out trade-offs, risks, and failure modes.
5. Propose validation and rollout steps.

## Output Expectations

Provide:
- Architecture overview
- Component responsibilities
- Trade-offs and risks
