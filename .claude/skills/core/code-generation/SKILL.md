# Code Generation

## Workflow

1. Read `AGENTS.md` and `CLAUDE.md` before edits.
2. Locate existing patterns and related files.
3. Implement the smallest change that meets the requirement.
4. Update or add tests if logic changes.
5. Run relevant checks when feasible.
6. Summarize changes and reasoning.

## Notes

- Preserve existing architecture and naming patterns.
- Avoid new dependencies unless required.

## Output Expectations

Report:
- Files changed
- Tests run (or why skipped)
- Any follow-ups
