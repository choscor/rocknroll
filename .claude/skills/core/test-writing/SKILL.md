# Test Writing

## Workflow

1. Identify the logic and expected behavior to cover.
2. Prefer existing test patterns and utilities.
3. Add tests for critical paths and edge cases.
4. Keep tests deterministic and isolated.
5. Run the minimal relevant test command.

## Output Expectations

Report:
- Tests added or updated
- Commands run
- Any gaps not covered
