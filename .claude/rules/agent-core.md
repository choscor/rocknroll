# Agent Core Rules

Applies to all sub-agents in this repository.

## Core Rules

1. Minimal changes: only modify files directly related to the task.
2. Follow project structure and architecture.
3. Preserve existing behavior unless explicitly required.
4. Add tests for new or changed logic when possible.
5. Explain major changes and reasoning.
6. Safe editing: prefer small, targeted edits over large rewrites.
7. Reuse existing utilities before adding new code.

## Recommended Flow

plan -> code -> test -> review
